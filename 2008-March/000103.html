This appears to cause a problem:

My config now has:

          {
            "type": "key",
            "map_to": "LEFTSHIFT+3",
            "keycode": 35,
            "repeat_freq": 0
          }


The # key is above my 3 on my wireless keyboard (it is US layout)

However

If I disconnect my wireless keyboard, my actual notebook keyboard is in
GBr layout.  If I switch gnome to this layout, the # key is by itself
above Enter, and above the 3 key, I have the £ symbol.

When my keyboard layout is in US mode, # on the phone inputs #.
When my keyboard layout is GBr, # on the phone gives £.

I can see that this is because you are inputting raw key data, not the
final character, but short of reading the gnome keyboard/language
layout, I am not sure how this can be overcome.
