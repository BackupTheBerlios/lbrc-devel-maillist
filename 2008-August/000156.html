I like your mockup, it is actually close to what I had in mind, so I
created my own mockup to show you what I actually have in mind. :)

What is not done yet is the new 'Actions' tab. I envision that as a
Treeview with 4 columns:
[Action name] [Handler] [Detail] [Cellphone action]
With:
Action name: The unique identifier of a certain action, used in the
profile tab to select the action from the (alphabetically sorted) list
of all actions combobox.

Handler: The event handler used for the action. E.g. XInput, DBus, etc.
Or, if the action consists of multiple events, it should say 'Multiple
events'. Only on double clicking the action (which opens the action edit
window) you see the list of event handlers and the details such as which
key pressed, what dbus command called, etc.

Detail: If action == multiple events again show 'Multiple events', else
show the details as explained above

Cellphone action: A list of signals that can be send to cell phone. As a
start these would be 'Presenter:Next slide', 'Presenter:Previous slide',
 'Presenter:Reset', 'Volume:Update'.

If a cell phone action is set, the applet should first check whether the
right cell phone view is set in the current profile to handle the
signal. If not, just ignore the cell phone signal. (E.g. if I have a
profile 'Presenter' and set the cell phone view to 'Normal', don't try
to signal 'Presenter:Next slide' to the phone if an action has that signal).

Jasper
